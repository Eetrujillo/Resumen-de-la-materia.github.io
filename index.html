Metodologías en Programación

Las metodologías son enfoques estructurados para gestionar el proceso de desarrollo de software. A continuación, te mostramos algunas de las más utilizadas:

1. Metodología Ágil (Agile)

La metodología ágil se enfoca en entregar resultados rápidamente a través de ciclos iterativos y colaborativos. En lugar de esperar a que todo el proyecto esté completo, se desarrollan pequeñas partes del proyecto y se presentan frecuentemente al cliente.

  • Iteraciones cortas: Sprints o ciclos de desarrollo cortos.
  • Colaboración constante: Trabajo continuo con el cliente.
  • Adaptabilidad: Cambio continuo según las necesidades.

2. Scrum

Scrum es un marco de trabajo dentro de Agile. Organiza el desarrollo en sprints que duran entre 1 y 4 semanas. El equipo se reúne diariamente para discutir el progreso y resolver cualquier impedimento.

  • Roles principales: Scrum Master, Product Owner y el equipo de desarrollo.
  • Reuniones clave: Sprint planning, daily stand-up, sprint review y sprint retrospective.

3. Cascada (Waterfall)

En esta metodología, el proyecto se divide en fases secuenciales. Cada fase debe completarse antes de pasar a la siguiente, lo que hace que sea menos flexible para adaptarse a cambios durante el desarrollo.

  • Fases: Requerimientos, diseño, implementación, pruebas, despliegue y mantenimiento.
  • Rígido: No permite cambios una vez que una fase ha comenzado.

Git y GitHub

Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar el código fuente y realizar cambios de forma controlada. GitHub es una plataforma que permite almacenar y colaborar en proyectos que usan Git.

Comandos Básicos de Git

  • git init: Inicializa un repositorio local en tu directorio.
  • git clone [url]: Clona un repositorio remoto en tu máquina local.
  • git add .: Añade todos los cambios al área de preparación.
  • git commit -m "mensaje": Registra un cambio en el repositorio.
  • git push: Sube los cambios al repositorio remoto.
  • git pull: Baja los cambios del repositorio remoto.
  • git branch [nombre]: Crea una nueva rama.
  • git merge [rama]: Fusiona una rama en la rama actual.

Ejemplo de un flujo de trabajo básico:

git init
git add .
git commit -m "Primer commit"
git remote add origin https://github.com/usuario/repositorio.git
git push -u origin master

GitHub - Pull Requests

En GitHub, después de hacer cambios en una rama, puedes abrir un pull request para solicitar que esos cambios sean revisados e integrados a la rama principal del repositorio. Este proceso permite la colaboración y revisión del código antes de fusionarlo.

Código HTML Básico

HTML (HyperText Markup Language) es el lenguaje estándar utilizado para crear páginas web. A continuación, se muestra la estructura básica de una página HTML.

Ejemplo de una página HTML básica:

<!DOCTYPE html>
<html>
    <head>
        <title>Mi Primera Página</title>
    </head>
    <body>
        <h1>Bienvenidos a mi página web</h1>
        <p>Este es un párrafo en HTML.</p>
        <a href="https://www.ejemplo.com">Ir a un sitio web</a>
    </body>
</html>

Este código crea una página simple con un título, un encabezado, un párrafo y un enlace.

A continuación se muestra el código Python para crear una base de datos llamada "autoconocimiento.db" y una tabla para el seguimiento del presupuesto:

import sqlite3 as sql

def createDB():
    conn = sql.connect("autoconocimiento.db")
    print("Base de datos de autoconocimiento creada")
    conn.close()

def createTable():
    conn = sql.connect("autoconocimiento.db")
    cursor = conn.cursor()
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS budget_tracking (
            budget_id INTEGER PRIMARY KEY AUTOINCREMENT,
            planning_id INTEGER,
            amount_spent DECIMAL(10,2) DEFAULT 0,
            amount_remaining DECIMAL(10,2),
            last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
            FOREIGN KEY (planning_id) REFERENCES experience_planning(planning_id)
        );        
    """)
    conn.commit()
    conn.close()
